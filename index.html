docker-desktop替换方案

发表于 2022-04-04

一、卸载docker-desktop软件
参考文档：https://www.thinbug.com/q/37465526
点击docker-desktop图标，选择：troebleshoot。

选择“Uninstall”，完成卸载。

卸载完成之后，发现docker命令也不存在了，下面开始安装docker-desktop的替代工具。

安装docker-desktop的替代方案
参考文档
https://arnon.me/2021/09/replace-docker-with-minikube/
https://yanbin.blog/replace-docker-desktop-with-hyperkit-minikube/#more-11697

相关替代方案：
1、dockercli+hyperkit+minikube
2、podman
3、lima：https://mdnice.com/writing/d6e0cdeac23a4bf29673b0bbd936737f
4、mircoK8s：https://microk8s.io/

方案一：dockercli+hyperkit+minikube
1、安装docker cli，因为之前卸载过docker所以重新安装docker cli


docker cli安装成功之后，发现docker engine并没有启动。

安装hyperkit

安装minikube，一个k8s集群快速搭建和管理的工具

启动k8s集群

minikube start –driver=hyperkit #(driver类型：docker，hyperkit,virtualbox)

遇到错误：由于之前driver指定的docker，现在需要更新为hyperkit，所以删除之前的配置：

minikube delete


错误2: 发现机器上没有安装docker-machine-driver-hyperkit，安装集群的时候，本身会下载这个工具，但是被墙下载失败，所以需要安装：

brew install docker-machine-driver-hyperkit
sudo chown root:wheel /usr/local/opt/docker-machine-driver-hyperkit/bin/docker-machine-driver-hyperkit
sudo chmod u+s /usr/local/opt/docker-machine-driver-hyperkit/bin/docker-machine-driver-hyperkit



重新安装docker-machine-driver-hyperkit，查看版本信息时，发现docker-machine-driver-hyperkiti不是一个二进制命令，是一个docker machine的插件，可以通过一下方式查看版本信息。

docker-machine version


重新执行：minikube start –driver=hyperkit，虽然还是会下载插件：docker-machine-driver-hyperkit。但是下载失败之后，不影响集群的启动（此时k8s 集群的相关镜像：minikube-v1.22.0.iso，在第一次执行minikue start时已经下载好了）。

下载kubernetes的客户端：kubectl

brew install kubernetes-cli
brew link kubernetes-cli
kubectl get pod -n kube-system


至此kubernetes集群已经搭建成功，但是docker ps 还是已经报错，没有连接到docker engine上。

minikube会启动docker-daemon的服务，在终端执行：eval $(minikube -p minikube docker-env)

在终端执行：docker ps ，可以看到当前运行的容器（同时其他命令也是可以使用，build，）。

每次打开终端是，默认初始化一些docker的环境变量，需要吧eval命令添加到:.zshrc文件中。

echo ‘eval $(minikube -p minikube docker-env)’ >> .zshrc

docker-desktop UI的替代品：kubernate dashboard

minikube dashboard –url

通过浏览器访问proxy的链接，里面展示了集群里所有的资源信息，比docker-desktop好用太多了。

但是docker pull 镜像的时候遇到报错：net/http: request canceled while waiting for connection (Client.Timeout exceeded while awaiting headers)
相关的issue：https://github.com/kubernetes/minikube/issues/4589
